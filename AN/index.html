Paso 1: Acceder al Modo EXEC Privilegiado

Permite pasar del modo de usuario (`Switch>`) al modo privilegiado (`Switch#`), necesario para ejecutar comandos de configuración y verificación avanzados.

Switch> enable

Paso 2: Acceder al Modo de Configuración Global

Desde el modo privilegiado, este comando ingresa al modo de configuración global (`Switch(config)#`), donde se pueden realizar cambios que afectan a todo el dispositivo.

Switch# configure terminal

Paso 3: Asignar un Nombre de Host

Cambia el nombre predeterminado del switch ("Switch") por uno más descriptivo ("S1") para identificarlo fácilmente en la red.

Switch(config)# hostname S1

Paso 4: Deshabilitar Búsquedas de DNS

Evita que el switch intente traducir comandos mal escritos como si fueran nombres de dominio, lo que ahorra tiempo al evitar pausas innecesarias.

S1(config)# no ip domain-lookup

Paso 5: Configurar el Mensaje del Día (MOTD)

Establece un banner o mensaje que se mostrará a cualquiera que intente acceder al dispositivo, usualmente para mostrar advertencias legales.

S1(config)# banner motd #Acceso no autorizado está estrictamente prohibido.#

Paso 6: Establecer Contraseña del Modo Privilegiado

Configura una contraseña secreta y encriptada ("class") para proteger el acceso al modo EXEC privilegiado. Es más segura que el comando `enable password`.

S1(config)# enable secret class

Paso 7: Proteger la Línea de Consola

Configura una contraseña para el acceso físico al switch a través del puerto de consola y exige que se solicite con el comando `login`.

S1(config)# line con 0 S1(config-line)# password cisco S1(config-line)# login

Paso 8: Proteger las Líneas Virtuales (VTY)

Aplica una contraseña a las líneas virtuales (0 a 15) para proteger el acceso remoto (Telnet/SSH). El comando `login` activa la solicitud de esta contraseña.

S1(config)# line vty 0 15 S1(config-line)# password cisco S1(config-line)# login

Paso 9: Encriptar las Contraseñas

Este servicio cifra las contraseñas que se guardan en texto plano (como la de consola y vty) en el archivo de configuración para mejorar la seguridad.

S1(config)# service password-encryption

Paso 10: Crear la VLAN de Administración

Crea una VLAN específica (VLAN 99) para la gestión del switch. Es una buena práctica de seguridad separar el tráfico de administración del tráfico de datos.

S1(config)# vlan 99

Paso 11: Ingresar a la Interfaz Virtual (SVI)

Accede al modo de configuración de la interfaz virtual de la VLAN 99 para poder asignarle una dirección IP.

S1(config)# interface vlan99

Paso 12: Asignar Dirección IP a la SVI

Configura una dirección IP y máscara de subred en la SVI para permitir la administración remota del switch a través de la red.

S1(config-if)# ip address 192.168.1.2 255.255.255.0 S1(config-if)# no shutdown
[ Imagen del Paso 12 ]

Paso 13: Configurar el Gateway Predeterminado

Establece la IP del router que el switch usará para comunicarse con dispositivos en otras redes. Es crucial para la administración remota.

S1(config)# ip default-gateway 192.168.1.1
[ Imagen del Paso 13 ]

Paso 14: Guardar la Configuración

Copia la configuración activa (running-config) a la memoria no volátil (startup-config) para que los cambios persistan después de un reinicio.

S1# copy running-config startup-config
